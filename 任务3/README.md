## 任务3（Tkinter 图形界面版）

本目录在 `任务2` 的基础上，升级为：

- `TCP` 信令控制：登录、呼叫、接听、拒绝、挂断、文本消息
- `UDP` 实时语音：每个音频帧携带 `sequence + timestamp`
- 接收端短时抖动缓冲：按序号放入缓冲队列，等待一个很短的播放延迟窗口后顺序播放
- 丢包补偿：默认不重传，优先用前后帧插值，回退到衰减重复帧
- 子网判断：同一子网优先 `P2P UDP` 直连，不同子网使用 `服务器 UDP 中转`
- 运行日志会明确显示当前通话使用的是 `P2P` 还是 `Relay`

### 运行方式

- 启动服务端 GUI
```bash
python server_gui_start.py
```

- 启动客户端 GUI
```bash
python client_gui_start.py
```

### 使用说明

1. 先启动服务端，再启动两个客户端。
2. 两个客户端分别输入各自昵称并连接到同一台服务器。
3. 主叫方输入目标昵称，点击“呼叫”。
4. 被叫方点击“接听”后，服务器会判断双方是否在同一子网：
   - 同子网：使用 `P2P UDP` 直连
   - 非同子网：使用 `服务器 UDP 中转`
5. 通话建立后，再点击“开始实时语音”发送麦克风数据。
6. 日志窗口会显示：
   - 子网判断结果
   - 当前链路模式（P2P / Relay）
   - UDP 探测、音频首包路径、丢包补偿情况
