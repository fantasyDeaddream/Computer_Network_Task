# 修复解散聊天室时客户端闪退问题

## 问题分析

解散聊天室时，服务器向所有成员发送 `room_dismissed_notify` 和 `room_member_update` 消息。创建者客户端的处理流程：

1. `dismiss_room()` 调用后，服务器广播消息
2. 客户端收到 `room_member_update`，触发 `_on_member_update`
3. `_on_member_update` 使用 `self.after(0, ...)` 异步调度 `_refresh_members`
4. 客户端收到 `room_dismissed_notify`，触发 `_on_dismissed` → `_handle_dismissed` → `_on_exit_room`
5. `_on_exit_room` 销毁 `ChatRoomFrame` 及其所有子控件（canvas 等）
6. Tkinter 事件队列中待执行的 `_refresh_members` 回调运行时，canvas 已被销毁
7. `canvas.delete("all")` 抛出 `TclError: invalid command name`，导致客户端闪退

## 解决方案

在 `_refresh_members` 方法开头添加控件有效性检查，如果控件已被销毁则直接返回。

### 修改文件

**client_gui.py** - `_refresh_members` 方法（约第 1176 行）

```python
def _refresh_members(self, members, positions):
    # 检查控件是否仍然有效（避免在 destroy 后执行导致 TclError）
    try:
        self.winfo_exists()
    except tk.TclError:
        return
    
    # ... 原有代码
```

### 原理

`winfo_exists()` 方法会检查 Tkinter 控件是否仍然存在。如果控件已被销毁，会抛出 `TclError`，捕获后直接返回，避免后续对已销毁控件的操作。
