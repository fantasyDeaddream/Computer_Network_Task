# 任务5 - 多方语音会议系统

这是在任务4基础上扩展出的多人语音会议系统。当前版本保留 `TCP + JSON` 控制面，同时把语音数据面改成了 `UDP + IP Multicast`。

## 当前播放方式

现在的多路语音不是“谁先到就直接播放谁”，而是按发送者分成多条独立音频路：

- 每个远端发送者都有自己的序号空间、时间戳和抖动缓冲区。
- 每路都先做冗余恢复和丢包补偿，再做逐路监测。
- 对检测出低频噪音的路，先做高通滤波抑制嗡声。
- 然后根据每路当前语音能量选出主发言人。
- 最后把所有路按“主发言人优先”的权重混音成一路，再统一送到扬声器播放。

## 音频链路要点

- 服务端只负责登录、联系人、邀请、入会/退会、解散会议等控制逻辑。
- 每个聊天室创建时，服务端分配一个组播地址和端口，并返回给客户端。
- 客户端通过 UDP 向该组播组发送音频，服务端不再做逐个成员音频转发。
- 每个音频包都带有：
  - 当前帧序号
  - 当前帧时间戳
  - 上一帧的冗余副本
  - 上一帧的时间戳
- 接收端默认不重传，主要依赖编码冗余、缓冲和插值来平滑播放。

## 新增能力

- 逐路低频噪音监测
- 逐路高通滤波
- 主发言人优先混音
- 每路音频时间戳保留，便于后续做跨路同步或对齐
- 监测快照接口：`ConferenceClient.get_audio_monitor_snapshot()`

## 主要文件

| 文件 | 说明 |
|------|------|
| `conference_protocol.py` | 控制协议定义与编解码 |
| `conference_server.py` | 服务端，负责 TCP 控制和组播地址分配 |
| `conference_client.py` | 客户端，负责逐路接收、监测、滤波、混音与播放 |
| `multicast_audio.py` | 音频传输辅助模块，包含包格式、时间戳、抖动缓冲、滤波和混音 |
| `test_core.py` | 控制面集成测试 |
| `test_multicast_audio.py` | 音频层单元测试 |

## 运行

启动服务端：

```bash
cd 任务5
python server_gui_start.py
```

或：

```bash
cd 任务5
python server_start.py
```

启动客户端：

```bash
cd 任务5
python client_gui_start.py

说明：

- 任务5 的控制面可以连 `localhost`，但音频面使用的是 IP 组播。
- 实际联调时更建议客户端连接服务端的局域网真实 IP，而不是只用 `localhost`。
```

## 测试

控制面测试：

```bash
python test_core.py
```

音频层测试：

```bash
python test_multicast_audio.py
```
