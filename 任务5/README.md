# 任务5 - 多方语音会议系统

基于局域网组播技术的多方语音会议系统，从任务4的"一对一"通话扩展为"一对多、多对多"。

## 功能特性

- **创建聊天室**：用户可在IP电话界面点击"创建聊天室"按钮创建语音会议室
- **邀请用户**：在聊天室中输入其他用户名称邀请他们加入
- **接受邀请**：被邀请用户在IP电话界面点击"接听"按钮加入聊天室
- **多方语音**：聊天室内所有人都能说话、所有人都能听见
- **退出/解散**：普通成员可退出聊天室，创建者可解散整个聊天室
- **最多20人**：每个聊天室最多容纳20个用户，以5x4网格展示

## 文件结构

| 文件 | 说明 |
|------|------|
| `conference_protocol.py` | 会议协议模块，定义消息类型和编解码函数 |
| `conference_server.py` | 会议服务器，管理聊天室和转发音频 |
| `conference_client.py` | 会议客户端，处理聊天室逻辑和音频收发 |
| `emodel.py` | 基于 ITU-T G.107 E-model 的通话质量计算与收包统计 |
| `client_gui.py` | 客户端图形界面 |
| `server_gui.py` | 服务端图形界面 |
| `tk_utils.py` | Tkinter工具函数 |
| `client_gui_start.py` | 客户端启动器 |
| `server_gui_start.py` | 服务端启动器（GUI版） |
| `server_start.py` | 服务端启动器（命令行版） |

## 使用方法

### 1. 启动服务器

```bash
cd 任务5
python server_gui_start.py    # GUI版
# 或
python server_start.py         # 命令行版
```

### 2. 启动客户端

```bash
cd 任务5
python client_gui_start.py
```

### 3. 使用流程

1. 启动服务器
2. 启动多个客户端，分别登录不同用户名
3. 用户A在"IP电话"页面点击"创建聊天室"
4. 在聊天室界面点击"邀请用户"，输入用户B的用户名
5. 用户B在"IP电话"页面看到邀请通知，点击"接听"加入
6. 所有成员可以进行多方语音通话
7. 普通成员可点击"退出聊天室"，创建者可点击"解散聊天室"

## 技术实现

- **信令通道**：TCP连接，JSON消息协议
- **音频转发**：服务器接收音频数据后转发给聊天室内所有其他成员（模拟组播）
- **端口**：默认使用8882端口（与任务4的8881区分）
- **复用代码**：登录逻辑、联系人管理、数据存储均复用任务4的实现

## E-model 通话质量评价

本任务已在多方通话中加入 ITU-T G.107 E-model 评价链路：

1. 客户端发送音频帧时附带 `seq` 序号和 `timestamp_ms` 发送时间戳。
2. 接收端按每个说话人统计丢帧率、平均时延和抖动。
3. `emodel.py` 使用简化 G.107 公式计算 R 值和 MOS：

```text
R = Ro - Is - Id - Ie_eff + A
MOS = 1 + 0.035R + R(R - 60)(100 - R) * 7e-6
```

其中 `Id` 由单向时延估算，`Ie_eff` 由音频帧丢失率估算。G.107 本身没有独立的抖动项，因此程序把抖动折算到有效时延中。聊天室界面底部会显示每个说话人的 R 值、MOS、时延、抖动、丢包率和质量等级。

注意：单向时延由发送端和接收端时间戳相减得到。若客户端运行在不同机器上，建议先进行系统时钟同步；否则 R 值仍可反映丢包/抖动趋势，但时延项会包含时钟偏差。

### 模拟不同网络损伤

服务端支持通过环境变量在转发音频时模拟网络损伤：

| 环境变量 | 含义 | 示例 |
|------|------|------|
| `TASK5_DELAY_MS` | 固定转发时延，单位 ms | `100` |
| `TASK5_JITTER_MS` | 随机抖动范围，单位 ms | `30` |
| `TASK5_LOSS_RATE` | 音频帧丢失率，可写 `0.05` 或 `5` 表示 5% | `0.05` |

PowerShell 示例：

```powershell
$env:TASK5_DELAY_MS="150"
$env:TASK5_JITTER_MS="40"
$env:TASK5_LOSS_RATE="0.05"
python server_start.py
```

启动多个客户端并进入同一个聊天室后，改变上述参数重启服务端，就可以比较无损、单纯时延、抖动、丢包以及组合损伤下的 R 值和 MOS 变化。
